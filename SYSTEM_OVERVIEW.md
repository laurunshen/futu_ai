# Futu Paper AI 系统总览

最后更新：2026-06-12

这份文档用于防止过段时间忘记“系统为什么这样设计、现在到底在跑什么、后面该看什么”。工程细节放在 `PROJECT_CONTEXT.md`，策略原则放在 `STRATEGY.md`，这份文件只写当前闭环、实验设计和后续方向。

## 一句话目标

这个项目不是让 AI 直接保证盈利，而是搭一个可记录、可比较、可复盘的交易学习系统：

- 用 Futu OpenD 提供行情和模拟交易环境。
- 用 autoNews 提供结构化新闻信号。
- 用 Gemini 把行情、新闻、持仓、现金、风格假设和风控参数合成决策。
- 用本地多组合账本记录不同策略盘的决策、成交、费用、收益、回撤和后续 1/3/7 天表现。
- 最终判断：哪些新闻、哪些策略、哪些仓位纪律在一段周期内真的更有用。

## 当前系统闭环

当前主链路是：

1. Futu OpenD 拉行情快照，包括常规价格、bid/ask、成交量、涨跌幅，以及美股盘前/盘后/夜盘情绪字段。
2. autoNews 从 SQLite 新闻库读取最近高影响新闻，按候选股、观察池和宏观锚点匹配。
3. Watchlist 按 `core` / `learning` / `opportunity` / `macro` 分层：
   - `core`：真实持仓、曾经持仓、高关注标的和直接竞争对手。
   - `learning`：想系统学习的行业样本，例如中概互联网、电商、AI 算力、半导体。
   - `opportunity`：可能参与波段机会发现的高流动性标的。
   - `macro`：指数、ETF、利率、美元、中概锚点，只做上下文，不参与买卖。
4. 候选筛选先看行情异动，再强制纳入：
   - 当前持仓。
   - 高影响新闻命中的标的。
   - 涨跌幅、量比、振幅、盘前盘后异动超过阈值的标的。
5. Gemini 使用 TradingAgents-lite 风格输出：
   - market_analyst
   - news_analyst
   - portfolio_analyst
   - bull_case
   - bear_case
   - risk_review
   - manager_summary
   - 最终 `BUY` / `SELL` / `HOLD`
6. 风控和交易时段检查在代码层执行：
   - 常规交易时段可以应用订单。
   - 开盘前准备窗口只扫描，不模拟成交。
   - 闭市、午休和周末不允许 AI/manual 决策生成本地成交。
   - `0` 表示该项风控未启用限制，不表示预算为 0。
7. 本地账本记录：
   - 决策日志 `data/decisions/YYYY-MM-DD.jsonl`
   - 组合账本 `data/state/portfolios.json`
   - 持仓、现金、费用、交易来源、外部现金流、FX 来源
8. 评估页追踪：
   - 组合收益曲线和最大回撤。
   - 资金流调整后的收益。
   - AI 决策后 1/3/7 天表现。
   - 新闻触发 vs 非新闻触发。
   - 交易频率、费用占比、应用状态、blocked 原因。

## 数据口径

已经修过且目前应维持的关键口径：

- 账本写入使用文件锁和原子替换，避免 web 与 loop 并发写坏 JSON。
- 现金注资/出金记录为 external cash flow，收益曲线同时保留原始 NAV 和资金流调整口径。
- 本地成交计入盈立/USMART 标准佣金和代收费，买入成本和卖出已实现盈亏都包含费用。
- FX 优先用富途 OpenD FX；不可用时用第三方汇率；组合可以保存手动券商校准汇率；所有来源都写入记录。
- 富途大资金盘如果没有持仓和交易，汇率变化不应被看成策略收益；现金-only 盘的绩效收益应校准为 0。
- 美股盘前/盘后/夜盘只作为情绪信号，不作为成交价格。
- AI 决策的 `BUY` 必须给正的 `max_notional`；如果没有明确金额，必须 HOLD。

## 当前组合层

### 实际仓位镜像

`l r s`

- 代表用户真实券商仓位的本地镜像。
- 用来复盘真实持仓、真实换仓和真实判断。
- AI 可以给建议，但不应把它称为教学模拟盘。

### 旧自动盘和富途验证盘

`l r s - Auto`

- 小资金自动实验盘。
- 与真实镜像相近，用来观察 AI 自动应用后的行为。

`富途大资金操作盘`

- 大资金验证盘。
- 用于观察富途模拟环境下的大资金购买力、成交回写和执行差异。
- 若无持仓和无交易，收益应为 0；不能让汇率波动污染收益判断。

## 当前实验纪律

开跑后可以修管道问题，但不要随意改实验变量。

可以随时修：

- 服务没跑。
- 日志没写。
- 订单应用异常。
- 评估页计算错误。
- FX、费用、收益口径 bug。
- 风控字段被模型误解。

不要中途改：

- Gemini 模型版本。
- 每个盘的 prompt_template。
- 每个盘的 strategy_hypothesis。
- 每个盘的交易节奏。
- 每个盘的单笔上限。
- Watchlist 大结构。

如果必须改这些，应该把它当成新策略版本，从修改日重新计时。

当前服务器模型：

```text
GEMINI_MODEL=gemini-3.5-flash
```

它是固定字符串，不是 `latest` 浮动别名。以后如果换模型，要在文档和策略假设里记录版本切换日期。

## 新闻维度到因子层的路线

当前不急着拆太细的因子。正确路线是：

1. 第一阶段先验证“新闻维度整体有没有信号”。
2. 如果新闻触发的决策明显优于无新闻触发，再进入新闻因子拆解。
3. 如果没有差异，不要硬拆因子，应先怀疑新闻源覆盖、时延、impact_score 和候选匹配。

第一阶段看两类问题：

- 盘级问题：B10/B30 新闻动量盘是否跑赢 A10、C10/C30、D10/D30。
- 决策级问题：所有组合里，高影响新闻触发的决策，在 1/3/7 天表现上是否优于无新闻触发的决策。

第一个天然因子是 `impact_score`：

- 把新闻触发决策按 impact_score 分桶。
- 看高分桶的 1/3/7 天命中率、平均收益和回撤是否优于低分桶。
- 如果 impact_score 和结果不相关，后续细分事件类型意义会下降。

第二阶段才拆：

- 宏观新闻：利率、通胀、战争、关税、汇率。
- 行业新闻：AI 算力、半导体出口限制、电商竞争、互联网监管。
- 个股新闻：财报、回购、订单、管理层、监管处罚。
- 情绪/技术新闻：异动、放量、盘前盘后跳变、板块共振。

样本量预期：

- 7 天只能看系统是否正常、B/C/D 的行为差异、交易频率、费用和 blocked 分布。
- 2-4 周可以初步看新闻维度是否有方向性。
- 因子层面的可信结论通常要 2-3 个月，因为一拆类型，每类样本会很少。

## 观察节奏

第 1-2 天：

- 看服务是否正常运行。
- 看每个 auto 盘是否都有决策日志。
- 看是否出现成片 blocked。
- 看有无订单、费用和收益口径异常。
- 看 Gemini 是否误解 prompt 或风控字段。

第 7 天：

- 主要看行为差异，不急着评收益胜负。
- 重点看 B 盘是否更频繁响应新闻。
- 看 D 盘是否比 C 盘交易更多、费用更高、是否重复加仓。
- 看新闻触发决策和非新闻决策的 1/3/7 天追踪是否有明显差别。

第 2-4 周：

- 初步看维度级结论：新闻维度是否值得继续。
- 看 C 系列规模梯度是否改善执行质量。
- 看自由盘是否真的提升收益，还是只制造噪声。

第 2-3 个月：

- 才适合进入新闻因子拆解和策略版本迭代。

## 每周复盘问题

每周至少问这些问题：

- 哪些盘有真实交易，哪些只是一直 HOLD？
- B 盘是否因为新闻触发而交易，还是也在宏观风险下持续 HOLD？
- D 盘的交易次数是否显著高于 C 盘？
- D 盘多出来的交易有没有带来扣费后收益？
- C10、C30、C50、C100 的差异是否来自资金规模，而不是 prompt 差异？
- 新闻触发决策的 1/3/7 天表现是否优于无新闻触发？
- impact_score 高的新闻是否真的更有效？
- 哪些 blocked 是合理风控，哪些是配置或代码问题？
- 有没有因为汇率、现金流、费用或闭市价格污染收益曲线？

## 下一批功能候选

不要现在全做，等第一批样本跑出来再决定优先级。

高价值候选：

- A10 固定建仓篮子，让它从现金对照变成真正买入持有基准。
- “我的预判”字段：用户先写判断，再看 AI 判断，用来训练自己的交易认知。
- 卖出/换仓反事实追踪：例如卖出 MSFT 后，继续追踪如果没卖会怎样。
- 慢扫描轮巡：按行业轮巡观察池，补足无新闻无异动股票的认知盲区。
- 新闻因子自动打标：在新闻维度有效后，再把历史标题批量标成事件类型。
- SQLite 迁移：当前文件锁已经足够安全；等数据量和查询复杂度上来后再迁。
- HTTPS：公网 Basic Auth 仍是 HTTP，后续可用 Caddy / Let's Encrypt。

## 目前最重要的心法

- 系统先看新闻维度，不急着拆因子。
- 先看行为，再看收益；先看 7 天机制，再看 30 天结论。
- 中途手痒想改策略，先写下来，不要直接改正在跑的实验盘。
- 模型输出是研究助理，不是神谕；真正的资产是不断积累的决策、结果和复盘标签。
- 认知来自“先判断、再验证、再复盘”，不是来自每天看更多股票。
